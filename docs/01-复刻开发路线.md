# 复刻开发路线

## 目标

本项目按照 `G:\fourSurvey\四普离线端分析文档包` 中已有分析材料推进复刻，采用“先打基础平台，再做核心业务闭环，最后补地图与报送”的顺序。

当前原则：

1. 先复刻业务闭环，不在第一阶段追求旧系统所有细节。
2. 先建立新库和新接口边界，再处理历史兼容和数据迁移。
3. 地图能力单独抽层，避免一开始把主业务和地图服务绑死。

## 阶段划分

### 阶段 1：基础平台

目标：建立可持续开发的骨架和最小通用能力。

范围：

- 工程结构与开发规范
- 用户、角色、组织、区域
- 基础字典
- 通用接口返回结构
- 通用列表与详情页骨架

本阶段完成标志：

- 可以用系统账号登录
- 可以获取当前用户信息、菜单和组织信息
- 可以维护基础字典与行政区划

### 阶段 2：核心数据建模

目标：把旧系统里的核心业务对象重新整理成可维护的新库结构。

范围：

- 普查对象主档
- 普查对象点位
- 参照底账
- 接收批次与接收记录
- 核查任务
- 附件资源

本阶段完成标志：

- 首批核心表结构确定
- 后端领域包和接口边界稳定
- 可以支撑详情页和列表页开发

### 阶段 3：核心业务闭环

目标：打通最关键的业务链路。

范围：

- 数据初始化
- 接收移动端数据
- 普查对象详情与编辑
- 质量核查

本阶段完成标志：

- 可以从基础底账开始形成普查对象
- 可以接收外业回传并入库
- 可以发起和处理核查任务

### 阶段 4：增强能力

目标：补齐旧系统里的高价值能力。

范围：

- 地图工作台
- 附件上传与导出
- 统计首页
- 报送省平台

本阶段完成标志：

- 地图能与业务对象联动
- 常用导出与统计可用
- 报送链路具备可验证的最小闭环

### 阶段 5：迁移与试点

目标：完成历史兼容、试点验收和切换准备。

范围：

- 旧库迁移脚本
- 字段映射校验
- 试点数据导入
- 验收测试

本阶段完成标志：

- 历史数据可按计划迁入新系统
- 试点单位完成业务验证
- 形成上线切换清单
