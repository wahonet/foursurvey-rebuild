# Four Survey Rebuild

## 项目目标

本目录用于复刻“第四次全国文物普查离线端”新系统源码骨架。

当前阶段目标：

1. 先搭建新系统工程结构
2. 先落基础平台和数据主干
3. 再逐步实现核心业务闭环

不在第一阶段追求：

- 全量页面复刻
- 地图能力完全兼容
- 全量模板与升级机制复刻

## 当前目录

- `backend`：Spring Boot + MyBatis 后端
- `frontend`：Vue 3 + Vite 前端
- `docs`：工程内简版说明
- `sql`：新库与迁移草案

## 技术路线

- 后端：Spring Boot 3.5 + MyBatis + MySQL
- 前端：Vue 3 + Vite + TypeScript
- 数据库：MySQL，按业务域重建
- 地图：后续独立抽层接入

## 开发顺序

### 第 1 步：基础平台

- 登录与权限
- 基础字典
- 通用列表页与详情页框架

### 第 2 步：核心数据建模

- 普查对象主档
- 参照底账
- 接收批次
- 核查任务

### 第 3 步：核心业务闭环

- 数据初始化
- 接收移动端数据
- 普查对象详情
- 质量核查

### 第 4 步：增强能力

- 地图工作台
- 附件与导出
- 报送省平台
